AppKit Headless allows developers to build a fully customizable ‚ÄúConnect‚Äù experience powered by AppKit‚Äôs existing features without using AppKit modal. Headless exposes wallet discovery, rendering WalletConnect QR, and social providers flow via simple hooks and utility functions.

## Installation

Headless requires AppKit to be configured with `enableHeadless` flag. See the [installation](../core/installation) docs before getting started and make sure you're enabling headless as expected.

```tsx
createAppKit({
  ...
  enableHeadless: true
})
```

## useAppKitWallets

The essential hook for accessing all type wallets and methods to build your custom connection user interface.

<CodeGroup>

```tsx useAppKitWallets.tsx
const { data, isFetchingWallets, isFetchingWcUri, wcUri, page, count, fetchWallets, connect } =
  useAppKitWallets()
```

</CodeGroup>

### Returns

<ResponseField name="data" type="WalletItem[]">
  List of all wallets (injected wallets and WalletConnect wallets combined).
</ResponseField>

<ResponseField name="isFetchingWallets" type="boolean">
  Boolean indicating if WalletConnect wallets are being fetched.
</ResponseField>

<ResponseField name="isFetchingWcUri" type="boolean">
  Boolean indicating if a WalletConnect URI is being fetched.
</ResponseField>

<ResponseField name="wcUri" type="string | undefined">
  The current WalletConnect URI for QR code display (set when connecting to a WalletConnect wallet).
</ResponseField>

<ResponseField name="page" type="number">
  The current page number of WalletConnect wallet results.
</ResponseField>

<ResponseField name="count" type="number">
  The total number of available wallets (for pagination/counting).
</ResponseField>

<ResponseField name="fetchWallets" type="(options) => Promise">
  Function to fetch WalletConnect wallets from the explorer API. Enables pagination/initial load.
</ResponseField>

<ResponseField name="connect" type="(wallet: WalletItem, namespace?: ChainNamespace) => Promise">
  Function to connect to a wallet. For WalletConnect wallets: starts connection & updates wcUri. For
  injected wallets: triggers direct connection.
</ResponseField>

### List and Connect to Injected Wallets

You could list the available injected wallet detected in your browser and connect to them. If you configure your AppKit as multi-chain, we suggest rendering a namespace selection component where users should select which namespace to connect. This could be achieved with the `connectors` array of the `WalletItem`.

<CodeGroup>

```tsx InjectedWallets.tsx
import React from 'react'
import { useAppKitWallets } from '@reown/appkit/react'

function InjectedWalletsList() {
  const { data } = useAppKitWallets()

  const injectedWallets = data.filter(wallet => wallet.isInjected)

  return (
    <>
      {injectedWallets.map(wallet => (
        <WalletItem key={wallet.id || wallet.name} wallet={wallet} />
      ))}
    </>
  )
}
```

```tsx WalletItem.tsx
export function WalletItem({ wallet }: { wallet: WalletItem }) {
  const { connect } = useAppKitWallets()

  async function handleConnect(wallet: WalletItem, namespace?: ChainNamespace) {
    await connect(wallet, namespace)
      .then(() => {
        toast({ title: 'Connected', status: 'success' })
      })
      .catch(() => {
        toast({ title: 'Connection declined', status: 'error' })
        setConnectingWallet(undefined)
      })
  }

  return (
    <div>
      <span>{wallet.name}</span>
      {wallet.connectors.map(item => {
        return <button onClick={() => handleConnect(wallet, item.chain)}>{item.chain}</button>
      })}
    </div>
  )
}
```

</CodeGroup>

### List and Connect to WalletConnect Wallets

WalletConnect supports 500+ wallets. You can let users to search their own wallets to connect to your app. Listing and searching WalletConnect wallets highly suggested for apps to give users a best connecting experience.

<CodeGroup>

```tsx WalletConnectWallets.tsx
const { data, isFetchingWcUri, isFetchingWallets, page, count, fetchWallets } = useAppKitWallets()
const [inputValue, setInputValue] = useState('')
const searchQuery = useDebounceValue(inputValue, 500)

// Filter wallets as non injected ones.
const wcWallets = data.filter(w => !w.isInjected)

useEffect(() => {
  fetchWallets?.()
}, [])

// Query WalletConnect wallets
useEffect(() => {
  if (searchQuery.length > 0) {
    fetchWallets?.({ query: searchQuery })
  } else {
    fetchWallets?.()
  }
}, [searchQuery])

return wcWallets.map(wallet => {
  return (
    <WalletItem
      name={wallet.name}
      onClick={() => {
        handleWallet(wallet)
      }}
      imageSrc={wallet.imageSrc}
      badge={wallet.type}
    />
  )
})
```

```tsx QRCode.tsx
const { wcUri, isFetchingWcUri } = useAppKitWallets()

if (isFetchingWcUri) {
  return <div>Getting the connection QR...</div>
}

if (wcUri) {
  return <QR value={wcUri} />
}
```
